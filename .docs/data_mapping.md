# V2 Data Schema & Integration Dictionary

This document explicitly maps out the schema requirements for Downstream Data ingested via the Read-Only QuickBooks Web Connector. If you are rebuilding the API in V2, you must strictly construct these schemas.

## Core Tenet
All integrated data lands dynamically in `/businesses/{tenantId}/[collection]`.

---

## 1. Customers (`qb_customers`)
Downstream payloads isolated where `Sublevel === 0`.
**V2 Schema:**
```json
{
  "listId": "string (QB ID)",
  "firstName": "string",
  "lastName": "string",
  "company": "string (Fallback to FullName)",
  "email": "string",
  "mobilePhone": "string",
  "workPhone": "string",
  "balance": "number",
  "status": "string ('Active' | 'Inactive')",
  "tags": ["QuickBooks"],
  "qbCustomFields": "object (Raw Key-Value pairs)"
}
```

## 2. Jobs (`qb_jobs`)
Downstream payloads isolated where `Sublevel > 0`.
Jobs inherit identically from Customers, with added specific operational fields derived computationally.
**V2 Schema Extension:**
```json
{
  ...CustomerSchema,
  "jobName": "string (Name field)",
  "parentRefId": "string (Parent ListID)",
  "vehicle": {
     "make": "Parsed from qbCustomFields",
     "model": "Parsed from qbCustomFields",
     "year": "Parsed from qbCustomFields",
     "vin": "Parsed from qbCustomFields"
  }
}
```

## 3. Inventory Items (`qb_items`)
Consolidated from `ItemInventoryQuery`, `ItemServiceQuery`, `ItemNonInventoryQuery`.
**V2 Schema:**
```json
{
  "listId": "string (QB ID)",
  "sku": "string (Name)",
  "name": "string (FullName)",
  "description": "string (SalesDesc)",
  "quantityOnHand": "number",
  "quantityAllocated": "number (Native to UpfittersOS)",
  "quantityOnOrder": "number",
  "cost": "number",
  "price": "number",
  "status": "string ('In Stock' | 'Inactive')",
  "type": "string ('Inventory' | 'Service' | 'NonInventory')"
}
```

## 4. Vendor Tracking (`qb_vendors`)
**V2 Schema:**
```json
{
  "listId": "string",
  "fullName": "string",
  "companyName": "string",
  "email": "string",
  "phone": "string",
  "balance": "number"
}
```

## 5. Staff Directory (`qb_employees`)
Tracks the master financial payroll identities strictly. Timelogs generated by UpfittersOS tie to this `listId`.
**V2 Schema:**
```json
{
  "listId": "string",
  "firstName": "string",
  "lastName": "string",
  "email": "string",
  "phone": "string"
}
```
